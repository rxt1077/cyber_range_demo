Challenge 3

From a backend perspective, this is the most complicated type of challenge. Instead of using a simple web page or a single container, it uses a group of containers via Docker Compose and allows the participant to VPN into the internal network. This simulates a full environment, but it's going to be pretty hefty in terms of compute requirements. You probably wouldn't want to do it for every challenge.
Use the following WireGuard configuration to connect to the network environment for this challenge: 
{{ wgconf }}
This will give you the IP address on the container network. There are two other containers on the network, one is your VPN server and the other has your flag.
